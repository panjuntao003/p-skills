# p-xhs：小红书一站式创作

账号定位：侦探奎因（前 PM → 独立开发者，用 AI 一个人做产品）。图文主攻。

## 流水线

```
选题评估 → 标题公式 → 文案写作 → 去AI味 → 配图方案
```

每步有独立的质量门，过不去就不往下走。

---

## Stage 1：选题评估

参考 `references/topic-selection.md`。核心：五个维度评分（长尾潜力 30%、可转视频 25%、争议/互动 20%、可验证性 15%、时效压力 10%），总分 50。≥40 直接做，30-39 确认后做，<30 跳过。

输出选题卡片：话题概括 + 评分表 + 3 个角度建议 + 标题方向 + 内容大纲。

---

## Stage 2：标题

参考 `references/title-formulas.md`。从 75 个在小红书验证过的爆款公式中匹配，12 类心理触发器：

| 类型 | 编号 | 适合 |
|------|------|------|
| 认知冲突 | 1-6 | 观点、科普、打破误区 |
| 好奇缺口 | 7-12 | 揭秘、行业内幕 |
| 恐惧/损失 | 13-20 | 避坑、警告 |
| 身份代入 | 21-25 | 垂直人群精准获客 |
| 数字锚定 | 26-32 | 方法论、清单 |
| 结果承诺 | 33-40 | 教程、变现 |
| 社会证明 | 41-47 | 经验分享、逆袭 |
| 争议/挑衅 | 48-55 | 观点、评论互动 |
| 场景/条件 | 56-60 | 解决方案 |
| 行动号召 | 61-66 | 态度、行动指南 |
| 权威借力 | 67-71 | 案例拆解 |
| 互动/测试 | 72-75 | 测评、投票 |

每条标题必须可追溯到具体公式编号。标题 ≤20 字（含标点），留悬念不说答案。

---

## Stage 3：文案

参考 `references/copywriting.md`。两步：

**Step A：诊断素材丰富度**
数据/故事/金句/权威/痛点逐项 0-10 分，总分：
- ≥30 → 写
- 15-29 → 警告后写
- <15 → 不写，回去补素材

**Step B：按规范写**

开头 = 话题 + Hook + 可信度。前 3 行决定打开率，独立建立吸引力。制造悬念，不给答案。

正文：每段 ≤3 行。"我"视角，有态度。每条事实注来源和核实时间。去掉一段后故事还完整的段就删。

结尾：留互动入口（问题/投票/预告）。

输出格式含 frontmatter（tags, date, platform, 来源, 核实）和来源核实清单。

---

## Stage 4：去 AI 味

6 项扫描：

1. **AI 起手词**：`其实/值得注意的是/简单来说/总的来说` → 去掉或换个人表达
2. **段末总结**：`这本身就是/从这里可以看出` → 删掉
3. **对比式铺垫**：`不是…而是…` → 只说"是"
4. **空洞形容词**：`非常/强大/优秀` → 用具体数据代替
5. **感叹号**：≤3 个
6. **emoji**：≤文案段数

**不改内容，只改表达。** 扫描 → 诊断（哪些最严重）→ 逐处修改 → 输出对比 + 完整版。

通用场景去 AI 味见 `references/humanizing.md`（24 种 AI 写作模式）。

---

## Stage 5：配图方案

参考 `references/image-planning.md`。

封面：1080×1440，3:4 竖版。纯色背景 + 大标题（≤7 字）+ 产品/工具截图。不用花哨拼贴。

内图：按阅读顺序规划每张图（步骤截图/代码/对比/数据），加简洁文字说明。

标准教程 6-9 张，简单分享 3-5 张，纯文字 1 张。

输出：封面设计 + prompt + 内图方案表 + 文件命名规范。

---

## 红线

- 素材不够（<15/50）不硬写
- 每条事实有来源和核实时间
- 不写需要实测才能验证的内容
- 不自问自答
- 不写结论式开头
