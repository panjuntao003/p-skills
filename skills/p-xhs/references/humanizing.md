# Stage 4：去 AI 味

## 小红书专用（6 项扫描）

对文案逐项扫描，每项命中 ≥ 2 处就算问题。

### 1. AI 起手词
`其实/值得注意的是/简单来说/总的来说/换言之/也就是说/不可忽视的是/值得一提的是`
→ 去掉，或换成个人表达（"我跟你说/说实话/讲真"）

### 2. 段末总结
`这本身就是…/从这里可以看出…/这一点说明了…/因此…`
→ 删掉，让读者自己得出结论

### 3. 对比式铺垫
`不是…而是…/并非…而是…/与其说…不如说…`
→ 只说"是"的部分，不否定前半句

### 4. 空洞形容词
`非常/强大/优秀/出色/显著/极佳/惊人/完美`
→ 用具体数据或效果代替。`效果显著` → `3 天涨粉 5000`。没数据就删。

### 5. 感叹号
≤ 3 个。超了就砍到 1-2 个。

### 6. emoji
≤ 文案段数。不要每句以 emoji 结尾，不要用 emoji 代替标点。

## 通用场景（24 种 AI 写作模式）

以下模式出现在任何中文文本中都应该被检测和修复：

### 夸大的象征意义
"标志着""见证了""充满活力""历史性的"→ 删或换成具体描述

### 宣传性语言
"重磅""颠覆性""革命性"→ 删，让事实自己说话

### 三段式法则
"第一…第二…第三…"→ 打破序号结构，用自然过渡

### AI 词汇
"赋能""抓手""闭环""颗粒度""底层逻辑"→ 换成具体说法的日常表达

### 过度连接
"此外""另外""与此同时""值得注意的是"→ 删掉大部分，中文靠意合

### 完美收束
段末、文末的总结升华句 → 删，留一点未完成感

### 英文直译腔
"在某种程度上""从某种意义上说"→ 删

### 形容词通胀
"非常重要的关键"→ "关键"。砍掉修饰词。

## 工作流程

扫描 → 诊断（最严重的 1-2 个问题）→ 逐处修改 → 输出对比 + 完整版

## 红线

- 不改内容（事实、观点、案例都不改）
- 不优化开头（那是文案写作的事）
- 不优化标题（那是标题公式的事）
- 不加虚构细节——真实感来自表达方式，不编故事
- 文本没扫描完之前不改——先诊断再修改
