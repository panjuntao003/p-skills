# Stage 3：文案写作

## 诊断素材丰富度

逐项检查：

| 维度 | 检查 | 评分 |
|------|------|------|
| **数据** | 有大数字、对比数字、百分比吗？ | /10 |
| **故事** | 有转变故事、反差案例吗？ | /10 |
| **金句** | 有独立成立的观点、可传播的记忆点吗？ | /10 |
| **权威** | 有人物、机构、报告背书吗？ | /10 |
| **痛点** | 有目标人群的焦虑、常见错误做法吗？ | /10 |

决策：
- ≥ 30/50 → 写
- 15-29/50 → 写但警告素材不足
- < 15/50 → 不写，回去补素材

## 开头公式

```
好开头 = 话题（讲什么）+ Hook（为什么看）+ 可信度（为什么信你）
```

输出 3-5 个版本，每个标注用了什么素材、制造了什么悬念。推荐 Top 1 并说明理由。

原则：
- 开头独立工作，不假设用户看了标题或封面
- 制造悬念，不给答案
- 用"为什么"不用"证明"，用"问题"不用"结论"
- "我"视角，有态度

## 正文规范

- 每段 ≤ 3 行
- "我"视角，有态度、有立场、有个人经验
- 事实性陈述标注来源和核实时间
- 去掉一个段落后故事还完整吗？如果是，删掉它

## 结尾

留互动入口，三选一：
- **问题**：「你觉得 XX 怎么样？」
- **投票**：「A 还是 B？」
- **预告**：「下一条讲 XX」

## 自检

| 检查项 |
|--------|
| 开头有钩子吗？有 Hook 和可信度吗？ |
| 每段超过 3 行了吗？ |
| 去掉一个段落后故事还完整吗？如果是，删掉它 |
| 每条事实数据都有出处并标注核实时间？ |

全部通过才输出。

## 输出格式

```markdown
---
tags: [小红书, 标签1, 标签2]
date: YYYY-MM-DD
platform: 小红书
type: 选题类型
来源: 核实记录
核实: YYYY-MM-DD
---
```
