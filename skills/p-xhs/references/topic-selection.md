# Stage 1：选题评估

## 评估模型

### 维度与权重

| 维度 | 权重 | 定义 |
|------|------|------|
| 长尾潜力 | 30% | 搜索量持续周期、是否过时快 |
| 可转视频 | 25% | 是否适合出镜讲、有画面感 |
| 争议/互动 | 20% | 能否引发评论区讨论 |
| 可验证性 | 15% | 能否核实关键事实（不编）|
| 时效压力 | 10% | 是不是过了今天就不值钱了 |

每维度 1-10 分，总分 50。

### 决策

| 总分 | 操作 |
|:----|:-----|
| ≥ 40 | 直接做。不需要确认。 |
| 30-39 | 报给用户确认。 |
| < 30 | 跳过。换下一个候选。 |

如果所有候选都 < 30：明确告知。

## 非共识检查

每个候选话题加一道检查：

```
- 这个话题对应的刻板印象是什么？
- 反例是什么？
- 新框架是什么？
- 新框架中需要定义的关键词是什么？
- 这个非共识是"正确的"吗？
```

只有能完整回答这 5 个问题的话题，才具备通过条件。

## 评估流程

```
原始信息
  │
  ▼
Step 1: 提炼核心话题（1-3句话说清楚这是什么事）
  │
  ▼
Step 2: 三维扫描
  │   ├ 搜索潜力：这个话题的天然搜索量？长尾关键词？
  │   ├ 视频潜力：能出镜讲吗？有视觉素材吗？
  │   └ 验证难度：我能核实到什么程度？
  │
  ▼
Step 3: 多角度切入
  │   对同一个话题列出3种不同的讲法：
  │   - 角度A：最安全（教程/科普向）
  │   - 角度B：最有争议（观点向）  
  │   - 角度C：最情感（故事/共情向）
  │
  ▼
Step 4: 输出选题卡片
```

```markdown
### 信息来源
- 来源链接：[URL]

### 话题提炼
这个事在讲什么：[2-3句话]

### 评分
| 维度 | 得分 | 理由 |
|------|------|------|
| 长尾潜力 | /10 | |
| 可转视频 | /10 | |
| 争议/互动 | /10 | |
| 可验证性 | /10 | |
| 时效压力 | /10 | |
| **总分** | **/50** | |

### 角度建议
**角度A（推荐）：** [一句话] — 为什么/适合人群/风险
**角度B：** [一句话] — 为什么/适合人群/风险

### 标题方向
- 方向1：[认知冲突 / 好奇缺口 / 等]
- 方向2 / 方向3

### 转视频思路
- 开头钩子 / 画面建议 / 结尾互动

### 做/不做？
✅ / ❌  理由：
```
